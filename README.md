## **Архитектура**

### C4-модель

    Контейнерный уровень архитектуры ПС
   <img width="445" height="488" alt="image" src="https://github.com/user-attachments/assets/3ee28317-b79d-49f1-9c8a-a6797bfb50e8" />
    Онлайн-сервис поиска автоперевозчиков в г. Минск представлен как программное средство CarDeal, которое обеспечивает взаимодействие между тремя основными внешними участниками: заказчиками автоперевозок, автоперевозчиками и оператор сервиса. Данные пользователи используют систему для решения своих задач: заказчики получают доступ ко всем карточкам автоперевозчиков, могут выбрать подходящего перевозчика по персональным параметрам, автоперевозчики добавляют свои услуги, а оператор сервиса занимается модератором контента в сервисе.
Система CarDeal включает несколько ключевых контейнеров. Основной элемент для взаимодействия пользователей с системой – это клиентское приложение, которое предоставляет интерфейс для удобной работы заказчиков и перевозчиков с сервисом, позволяя им формировать и просматривать заявки, управлять услугами.
Для обработки бизнес-логики и управления данными используется API-приложение, построенное на Spring Framework и развернутое на Apache Tomcat. Этот контейнер отвечает за приём запросов от клиентского приложения, их обработку и выполнение бизнес-правил, связанных с заказами перевозок и управлением пользователями.
Хранение данных реализовано в базе данных MySQL, которая содержит всю необходимую информацию об услугах, пользователях, перевозчиках и т.д. Доступ к базе данных осуществляется только через API-приложение, что обеспечивает централизованный контроль за чтением и записью данных, а также целостность информации в системе.
   Компонентный уровень архитектуры ПС
    <img width="624" height="394" alt="image" src="https://github.com/user-attachments/assets/99b5ea19-04cb-4f4e-93a6-f45b0cde5255" />
	На компонентном уровне система CarDeal детализируется как набор логически выделенных модулей внутри API-приложения, каждый из которых реализует отдельную часть бизнес-логики.
Клиентское приложение формирует пользовательский интерфейс и отвечает за отправку запросов к серверу для работы с системой.
На стороне сервера реализованы несколько специализированных компонентов (контроллеров):
1 AuthController – отвечает за регистрацию и авторизацию пользователей, а также за валидацию и выдачу JWT-токенов для безопасного доступа.
2 ReviewController – управляет отзывами, позволяя заказчикам оставлять оценки перевозчикам.
3 AdminController – предоставляет возможности модерации и управления системой (например, контроль пользователей).
4 RequestController – обрабатывает работу с заявками, включая их создание, редактирование и просмотр.
5 CarrierController – предоставляет автоперевозчикам инструменты для управления своими услугами и взаимодействия с заказчиками.
Все контроллеры взаимодействуют с базой данных MySQL, которая хранит ключевую информацию о пользователях, заявках, услугах и отзывах. Доступ к данным осуществляется централизованно через контроллеры, что обеспечивает целостность и безопасность работы системы.
Таким образом, на компонентном уровне C4-модели видно, что система CarDeal построена по модульному принципу: каждый контроллер отвечает за свою область ответственности, а взаимодействие между клиентом, сервером и базой данных реализуется через чётко определённые запросы и ответы
